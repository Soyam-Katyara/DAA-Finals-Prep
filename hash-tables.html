Direct Address & Hash Tables

Direct Address Table
Direct Address Table is a data structure that has the capability of mapping records to their corresponding keys using arrays. Records are placed using their key values directly as indexes. They facilitate fast searching, insertion and deletion operations.

✓ Advantages

  1. Search in O(1): Arrays are random-access; key = index, so lookup is instant.
  2. Insert in O(1): Place the value at table[key] directly.
  3. Delete in O(1): Set table[key] = null — done.

✗ Limitations

  • Requires prior knowledge of maximum key value.
  • Practical only when max key value is small.
  • Wastes memory when stored records ≪ table size.

What is a Hash Table?

A Hash Table is a data structure that maps keys to values using a hash function. Instead of storing data at the key's value directly (like Direct Address Tables), it computes an index using:

index = hash(key) mod table_size

This allows a large key space to map into a small array, saving memory. The trade-off: two different keys may hash to the same index — this is called a collision.

There are two main strategies for handling collisions:

Open Addressing

When a collision occurs, probe for the next available slot in the array itself. Techniques: Linear Probing, Quadratic Probing, Double Hashing.

Separate Chaining

Each slot holds a linked list. Colliding elements are simply appended to the list at that index.

Load Factor

The load factor of a non-empty hash table is the number of items stored divided by the size of the table.

λ = number of elements / hash table size

This is the decision parameter used when we want to rehash or expand the existing hash table.

Generally the threshold value is 0.75. So, if there are 12 values in a hash map of size 16, the load factor becomes 0.75 (16 × 0.75 = 12), and the table should be doubled in size.

Complexity of Hash Tables
Operation Best Case Average Case Worst Case Space
Search O(1) O(1) O(n) O(n)
Insert O(1) O(1) O(n) O(n)
Delete O(1) O(1) O(n) O(n)
Best / Average O(1): With a good hash function and low load factor, most operations hit the right slot immediately.

Worst O(n): In the worst case (e.g., all keys hash to the same slot), performance degrades to linear — equivalent to scanning a linked list or probing through every cell.

Space O(n): Hash table uses memory proportional to the number of stored elements.
